# スコアリング進化構想

2026年3月

---

## 1. 主観フィードバック — 「今日は昨日より良かった？」

### コンセプト

毎日LINE/Discordでプッシュ通知を送り、「今日は昨日より良かった？」にyes/noで回答してもらう。回答は任意。

### 設計原則

- **入力ゼロのコンセプトを壊さない**: 客観データ（Last.fm等）だけでシステムは完全に成立する。主観データは精度を上げる補助レイヤー
- **未回答ペナルティなし**: 答えなくてもスコアに悪影響はない。欠損として扱う
- **蓄積で精度向上**: 回答が多いほど客観スコアと主観感覚の相関分析が可能になる

### 活用方法

- 客観スコアと主観回答の乖離を検出（「スコアは高いのに本人はno」→ 指標の見直しシグナル）
- 主観データの蓄積により、基準値やカテゴリの重み付けを個人に最適化
- 長期的にはスコアリングモデル自体の精度検証に使える

### 実装メモ

- 既存の通知基盤（LINE Bot）を流用可能
- 回答は `raw_data` に `source=self_report`, `category=subjective` として保存
- score_method は既存のものとは別枠（スコアに直接加算するのではなく、メタデータとして保持）

## 2. パーセンタイルベースの自動基準値最適化

### 課題

現在の基準値は手動設定。新しいソースを追加するたびに「いい感じの基準値」を勘で決めている。ソース間のスケールも大きく異なる（Last.fmは週数百トラック、読書は月数冊）。

### アイデア

過去全記録の最高値・最低値から上位/下位パーセンタイルを算出し、基準値の自動設定やソース間スケーリングに活用する。

### 想定アプローチ

- 例: 上位25パーセンタイル = 「調子がいい時の水準」、下位25パーセンタイル = 「落ちている時の水準」
- この範囲を使って基準値を自動提案、あるいは手動設定の妥当性を検証
- ソース間で「自分にとっての上位/下位」を揃えることで、異なるスケールの指標を公平に比較可能

### 検討事項

- 初期データが少ない段階では信頼性が低い（最低N日分のデータが必要）
- 季節性やライフスタイル変化で分布自体がシフトする可能性
- 手動基準値との併用: 自動提案を参考に手動で確定、あるいは完全自動モード

## 3. イベントスコアの前後分配

### 課題

現在、イベント指標（ライブ、映画鑑賞等）はイベント発生日にスコアが一括加算される。これによりグラフが尖りやすく、実際の「文化的豊かさ」の感覚とずれる。

### アイデア

イベントのスコアをイベント当日だけでなく、前後の期間に分配する。

**根拠**: 楽しみに向かって生きている状態も十分に「文化的」である。ライブの3日前からワクワクしている状態は、文化的生活が豊かであることの表れ。

### 想定アプローチ

- イベント前: カレンダーに予定があるイベント（gcal_live等）は、前方にもスコアを分配
  - 例: ライブ当日100% → 当日50% + 前3日に各10% + 後3日に各~7%
  - カレンダー連携済みのソース（gcal_live, gcal_private）は未来の予定が見えるので前方分配の根拠データがある
- イベント後: 余韻としてのスコア分配（これは指数減衰で既に部分的に実現）
- 分配カーブは正規分布的（当日がピーク、前後に減衰）

### メリット

- グラフが滑らかになり、日単位の尖りが減る
- 応答性は維持（イベント当日に最大のスコアが付く）
- 「予定がある生活」自体を文化的活動として評価できる

### 検討事項

- 前方分配はカレンダー連携ソースのみ可能（Last.fmの再生やFilmarksの鑑賞には適用しにくい）
- 分配比率の設計（当日に何%残すか）
- 既存の指数減衰（decay_half_life）との関係整理
